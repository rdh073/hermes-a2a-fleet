# Code of Conduct

## TL;DR

Attack code, not people. Be technically honest. Don't be a jerk for sport.
That's it.

## Standards

**Acceptable:**
- Saying "this code is wrong because X" with reasoning
- Saying "I disagree, here's why" with reasoning
- Direct, blunt technical feedback
- Pointing out mistakes — yours, mine, anyone's
- Strong opinions held loosely; change your mind when the evidence changes

**Not acceptable:**
- Personal attacks ("you are stupid" — no; "this code is stupid" — fine if
  followed by reasoning)
- Harassment, threats, doxxing, sexual content, slurs
- Sustained disruption: derailing threads, ignoring maintainer decisions
  after a clear ruling, sealioning
- Retaliating against someone for filing a security report or bug

## How disputes are resolved

1. **Technical disputes** are resolved by argument quality. If you think a
   maintainer is wrong, show code, benchmarks, or a failing test. Not vibes.
2. **Conduct disputes** are resolved by the maintainer. Email
   <257243234+rdh073@users.noreply.github.com> with the issue/PR link and what happened. You'll
   get a response within 7 days.
3. **Repeat offenders** get warned, then temporarily blocked, then
   permanently blocked. The maintainer decides; the maintainer's decision
   is final.

## Scope

This applies to GitHub issues, PRs, discussions, the project chat (if any),
and any space where you're representing the project.

It does NOT apply to:
- Your personal social media
- Unrelated technical disagreements elsewhere
- Disagreeing with a maintainer's technical decision (that's allowed and
  encouraged — just keep it about the code)
